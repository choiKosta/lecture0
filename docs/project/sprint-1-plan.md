# Sprint 1 Plan

Timeline: 2 weeks (Sprint 1)

Scope (selected from `docs/requirements/productbacklogs/client-product-backlog.md`):

- CL-1: Server 접속 정보 입력/선택
- CL-2: Server 접속 기능
- CL-3: 접속 상태 UI 표시
- CL-4: RTSP/H.264 스트림 수신 및 표시
- CL-8: CMake 기반 빌드 설정 (빌드 검증)

Objectives:

- Deliver a working Client build that can connect to a test Server and display an RTSP stream.
- Provide basic UI for server connection and status.
- Establish CI smoke test for build verification.

Owners & Roles:

- Scrum Master: oversee sprint progress and impediments
- Client Developer: implement CL-1..CL-4, CL-8
- Product QA: prepare and run integration & acceptance tests
- Reviewer: code review and PR approvals

Definition of Done (for Sprint 1):

1. `cmake -S client -B client/build` and `cmake --build client/build` succeed.
2. Client can be configured with Server URL and successfully connect to RTSP test stream.
3. UI shows connection state and handles failure cases.
4. Basic integration smoke test runs (`ctest` or script) and reports pass/fail.
5. All implemented work has PRs referencing backlog items and assigned reviewers.

Daily cadence:

- Daily standup (short): status, blockers, plan for next 24h.
- Scrum Master collects impediments and escalates to Server Developer if interface issues arise.

QA gating:

- Product QA runs integration tests at mid-sprint and pre-release; failures yield GitHub Issues with `area:server` or `area:client` as appropriate.

Reporting:

- Progress updates posted to `client/docs/sprint-1-status.md` (generated by Scrum Master).
